FILE: backend/src/main/resources/db/migration/V1__create_users.sql

Content:
```sql
-- Phase 1: Users table
-- IMMUTABLE — do not modify after first apply. Create Vn migration for corrections.

CREATE TABLE users (
    id          BIGSERIAL       PRIMARY KEY,
    email       VARCHAR(255)    NOT NULL UNIQUE,
    password    VARCHAR(255)    NOT NULL,
    role        VARCHAR(50)     NOT NULL,
    full_name   VARCHAR(255),
    created_at  TIMESTAMPTZ     NOT NULL DEFAULT NOW(),
    updated_at  TIMESTAMPTZ
);

CREATE INDEX idx_users_email ON users (email);
```

Column notes:
- id: BIGSERIAL maps to @GeneratedValue(strategy = IDENTITY) in the JPA entity
- email: UNIQUE constraint; Hibernate validates this via ddl-auto=validate (the constraint must exist)
- password: VARCHAR(255) — sufficient for BCrypt output (60 chars max)
- role: VARCHAR(50) stored as a string enum name (ADMIN / OPERATOR)
- full_name: nullable — not required for auth
- created_at: server-side DEFAULT NOW(); entity @PrePersist also sets it as fallback
- updated_at: nullable; set by the service layer on updates
