<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| .env file → Docker Compose | Secrets loaded from host filesystem into container environment |
| application.properties → JVM | All secrets read from env vars; no hard-coded values |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | .env file | mitigate | .gitignore excludes .env; .env.example (no real secrets) is committed instead |
| T-01-02 | Information Disclosure | application.properties | mitigate | All secrets use ${VAR} placeholders; no defaults for JWT_SECRET |
| T-01-03 | Tampering | docker-compose.yml | accept | Local dev only; no production secrets in Compose file |
</threat_model>
