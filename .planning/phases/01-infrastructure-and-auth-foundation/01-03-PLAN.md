FILE: backend/src/main/java/com/opspilot/security/JwtService.java
- Annotations: @Service, @Slf4j (Lombok)
- Fields injected from properties:
  - @Value("${jwt.secret}") private String jwtSecret
  - @Value("${jwt.expiration-ms:3600000}") private long expirationMs
- Private method signingKey(): Keys.hmacShaKeyFor(Decoders.BASE64.decode(jwtSecret))
- public String generateToken(UserDetails userDetails):
  - Jwts.builder().subject(userDetails.getUsername()).issuedAt(new Date())
    .expiration(new Date(System.currentTimeMillis() + expirationMs))
    .signWith(signingKey()).compact()
- public String extractUsername(String token): parseClaims(token).getSubject()
- public boolean isTokenValid(String token, UserDetails userDetails):
  - extractUsername(token).equals(userDetails.getUsername()) && !isTokenExpired(token)
- private Claims parseClaims(String token):
  - Jwts.parser().verifyWith(signingKey()).build().parseSignedClaims(token).getPayload()
  - Uses parseSignedClaims() — NOT the deprecated parseClaimsJws()
- private boolean isTokenExpired(String token): parseClaims(token).getExpiration().before(new Date())

FILE: backend/src/main/java/com/opspilot/security/JwtAuthFilter.java
- CRITICAL: NO @Component annotation — the class must not be registered as a servlet filter by Spring Boot
- @RequiredArgsConstructor (Lombok)
- extends OncePerRequestFilter
- Constructor fields: JwtService jwtService, UserDetailsService userDetailsService
- Override doFilterInternal(HttpServletRequest request, HttpServletResponse response, FilterChain filterChain):
  1. Read Authorization header; if null or not starting with "Bearer ", call filterChain.doFilter() and return
  2. Extract token: authHeader.substring(7)
  3. Extract username: jwtService.extractUsername(token) — wrap in try/catch JwtException; on exception log warn and continue filter chain
  4. If username != null AND SecurityContextHolder.getContext().getAuthentication() == null:
     a. Load UserDetails: userDetailsService.loadUserByUsername(username)
     b. If jwtService.isTokenValid(token, userDetails):
        - Create UsernamePasswordAuthenticationToken(userDetails, null, userDetails.getAuthorities())
        - Set details: new WebAuthenticationDetailsSource().buildDetails(request)
        - CRITICAL — Spring Security 6 strategy-aware setter:
          SecurityContextHolder.getContextHolderStrategy().getContext().setAuthentication(authToken)
  5. filterChain.doFilter(request, response)

FILE: backend/src/main/java/com/opspilot/service/AuthService.java
- @Service, @RequiredArgsConstructor, @Slf4j
- Inject: AuthenticationManager authManager, JwtService jwtService, UserRepository userRepository
- public AuthResponse login(LoginRequest request):
  1. try { authManager.authenticate(new UsernamePasswordAuthenticationToken(request.email(), request.password())) }
     catch (BadCredentialsException ex) {
       log.warn("AUTH login failure: email={}", request.email());
       throw ex;  // re-throw; GlobalExceptionHandler maps to 401
     }
  2. On success: generate token from UserDetails obtained via loadUserByUsername or from authenticated principal
  3. Fetch the User entity: userRepository.findByEmail(request.email()).orElseThrow()
  4. log.info("AUTH login success: email={} role={}", user.getEmail(), user.getRole())
  5. return new AuthResponse(token, user.getEmail(), user.getRole().name())

- public UserResponse getCurrentUser(String email):
  - User user = userRepository.findByEmail(email).orElseThrow(() -> new UsernameNotFoundException(email))
  - return new UserResponse(user.getId(), user.getEmail(), user.getRole().name(), user.getFullName())

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client → AuthController | Untrusted credentials arrive at POST /api/auth/login |
| JWT → JwtAuthFilter | Token from Authorization header parsed for every protected request |
| SecurityContext → Controller | Principal propagated from filter to controller handler |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | JwtAuthFilter / JwtService | mitigate | JJWT verifyWith() enforces HS256 signature; parseSignedClaims() rejects alg=none and unsigned tokens |
| T-03-02 | Information Disclosure | JwtService | mitigate | JWT_SECRET read from env var only; never defaulted in application.properties; never logged |
| T-03-03 | Information Disclosure | AuthService | mitigate | log.warn logs email only on failure; password never logged or returned in any response |
| T-03-04 | Spoofing | AuthController POST /login | mitigate | @Valid @NotBlank prevents empty credential submission; AuthenticationManager verifies against BCrypt hash |
| T-03-05 | Elevation of Privilege | SecurityConfig | mitigate | /api/admin/** requires hasRole("ADMIN"); all other non-public endpoints require authenticated(); stateless session prevents session fixation |
| T-03-06 | Elevation of Privilege | Brute-force on /api/auth/login | accept | Rate limiting not in scope for Phase 1; documented as known limitation in CLAUDE.md constraints |
| T-03-07 | Information Disclosure | GlobalExceptionHandler | mitigate | 401 responses return generic "Invalid credentials" without disclosing whether email exists; stack traces not included in responses |
</threat_model>
